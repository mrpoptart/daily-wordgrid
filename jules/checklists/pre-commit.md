# Pre-Commit Checklist

Before submitting my work, I will run through the following checks to ensure a high-quality contribution.

## 1. Code Quality
- [ ] Code is formatted correctly and adheres to the project's style guide.
- [ ] Code is linted, and there are no linting errors.
- [ ] Code is well-documented with comments where necessary.
- [ ] All new dependencies have been added to the project's dependency management file.
- [ ] **Build Check:** I have run `npm run build` and confirmed that the build succeeds without errors.

## 2. Testing
- [ ] **Test Check:** I have run `npm test` and confirmed that all existing and new tests pass.
- [ ] New unit or integration tests have been added to cover the changes.

## 3. Verification
- [ ] I have read through my changes to catch any typos or logic errors.
- [ ] I have run the application locally (if applicable) to ensure the changes work as expected.

## 4. Documentation
- [ ] The `README.md` or other relevant documentation has been updated.
- [ ] The pull request description is clear, concise, and links to the relevant issue.
- [ ] The commit message is descriptive and follows the project's conventions.
